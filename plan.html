シールド線の接地：RPi 側の GND に片端のみ接続。両端接地するとグランドループになりノイズを拾う。

バイメタルサーモスタット：RPi 非依存の電源直結。AWG28 は電流容量不足のため対象外。ファン電源線（AWG18〜20）を使うこと。

ACC 連動走行前チェックシステム（Phase 4）

ACC 電源 ON を検知し、走行前に収納忘れ・開けっぱなしを自動チェック。異常があれば圧電ブザー + LED + ntfy.sh で警告。
センサー数: ソーラー×2（チルト+スライド）/ ジャッキ×4（前後左右）/ 窓×3（リア+サイドF+サイドR）/ 他4 = 合計13個以上
Codex (gpt-5.5) レビュー反映: ACC 検知はフォトカプラ絶縁必須。デバウンス・抑制込みの設計。

ACC 検知回路（Codex 推奨）:
ACC 12V → 1kΩ抵抗 + TVSダイオード → フォトカプラ（PC817）LED → GND
フォトカプラ出力: 3.3V → 4.7kΩプルアップ → GPIO
抵抗分圧のみは禁止。車載サージ（最大80V超）でGPIO破壊の危険あり。
動作仕様:
① ACC ON 検知 → 1〜3秒待機（起動ノイズ除去）
② 全センサースキャン（GPIO HIGH = 開放 = 異常）
③ 異常あり → ブザー周期鳴動 + LED 点滅 + ntfy.sh 送信
④ 走行中は 5秒間隔で再チェック。同一異常の ntfy 再送は 10分間隔で抑制
⑤ センサー線断線もHighになるためフェイルセーフ設計（誤判定注意。デバウンス 300ms 確定）
配線注意: センサー線をインバーター・DC-DC・モーター配線に近接させない。ルーフ等長距離配線はコルゲートチューブ保護 + 防水コネクタ（IP67以上）必須。シールド片端接地。

LED 照明制御（Phase 4）

① 車内メインライト — PWM 調光サイクル

タッチごとに状態が1段階進む: 切 → 弱（30%）→ 強（100%）→ 弱（30%）→ 切
スマホ（Flask / MQTT）からも同じ state 変数を操作して連携。
モジュール型番設定・接続
タッチセンサーTTP223MODEピン = オープン（モーメンタリ）→ GPIO 入力（立ち上がりエッジ割り込み）
MOSFETIRLZ44NGPIO12（ハードウェア PWM）→ ゲート。GND 共通化必須
PWM 設定—周波数 1kHz 以上（ちらつき防止）。duty: 切=0% / 弱=30% / 強=100%
LED24V 対応テープ / パネルヒューズボックスから直給電。RPi は MOSFET でスイッチのみ

② スライドドア足元ライト — 人感自動点灯

モジュール型番設定・接続
人感センサー（PIR）AM3123.3V 直接駆動可。検知距離 3m / 検知角 100°。出力 → GPIO 入力
MOSFETIRLZ44NGPIO → ゲート。GPIO High で LED ON
LED24V 対応テープ（足元用）ドア開放時 + 人感検知で点灯。消灯ディレイ（例: 30秒）はソフトウェアで設定
共通注意: MOSFET の GND と RPi の GND を必ず共通化。浮いていると誤動作する。
